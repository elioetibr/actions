# Bun Partial Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace pnpm (package manager) and Jest (test runner) with Bun while
keeping Vite for building, fixing ESM-only `@actions/*` package incompatibility
with Jest's CJS mode.

**Architecture:** Two sequential commits — Commit 1 swaps pnpm→bun as the
package manager (CI validates before touching tests); Commit 2 swaps jest→bun
test (removes Jest, adds `@types/bun`, creates `bunfig.toml`). No test files
change — Bun injects `jest.*` globals at runtime and `@types/bun` types them
identically.

**Tech Stack:** Bun 1.3.11, Vite 8, TypeScript 6, `@types/bun`, `bunfig.toml`,
`oven-sh/setup-bun@v2`

### Task 7: Replace jest.config.cjs with bunfig.toml

**Files:**

- Delete: `jest.config.cjs`
- Create: `bunfig.toml`

- [ ] **Step 1: Delete jest.config.cjs**

```bash
rm jest.config.cjs
```

- [ ] **Step 2: Create bunfig.toml at the repo root**

```toml
[test]
coverageThreshold = { line = 0.95, function = 0.95, branch = 0.95 }
coverageSkipTestFiles = true
```

Notes:

- Bun uses `0–1` range for thresholds (not `0–100` like Jest)
- `coverageSkipTestFiles = true` excludes `*.test.ts` files from coverage
  collection — equivalent to Jest's exclusion patterns
- Coverage is opt-in via `--coverage` flag; not enabled here so `bun test` stays
  fast

- [ ] **Step 2: Run tests with coverage**

- Exit 0

If coverage thresholds fail, the exit code will be non-zero and Bun prints which
metric failed.
